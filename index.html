CSS Slider
Mas como funciona?

Ótima pergunta. Vamos primeiro dar uma olhada na seguinte pseudo classe css:

elemento:target { /**/ }

Por definição, o :target representa o elemento cujo id combina com o identificador da url atual da página.
Ou seja, no momento em que sua url for:

http://url.com/#id1

O elemento que tiver o id = id1 se enquadrará no seletor.
O que nos permite fazer o seguinte: 

 span {
  color: black;
}

span:target {
  color: red;
}
Nesse cenário todos os span terão a cor preta, exceto aquele que tiver o id = id1, que terá a cor vermelha.
Agora vamos dar uma olhada no seguinte seletor:

elemento1 ~ elemento2 { /**/ }

Por definição, o elemento1 ~ elemento2 considera um elemento2 que for precedido por um elemento1 e que compartilha o mesmo elemento pai.
Difícil de entender? Vamos usar um exemplo então: 
 p ~ span {
  color: red;
}

<div>
  <span>Eu não sou vermelho.</span>
  <p>Eu sou um parágrafo.</p>
  <span>Eu sou vermelho!!!!!</span>
</div>
<span>Eu não sou vermelho.</span>
O que estamos dizendo é que queremos todos os span que venham depois de um p e pertençam ao mesmo pai de um p.

Ótimo! Agora já sabemos como usar o :target e o ~ :D
Já começou a imaginar como eles funcionam juntos pra fazer o slider?
Vamos ver agora:

Considere a seguinte estrutura: 
 <div class="slider">
    <div class="context-button" id="slide1"> </div>
    <div class="context-button" id="slide2"> </div>
    <div class="context-button" id="slide3"> </div>
    <figure class="slide">
        slide 1
    </figure>
    <figure class="slide">
        slide 2
    </figure>
    <figure class="slide">
        slide 3
    </figure>
    <div class="buttons">
        <a href="#slide1"></a>
        <a href="#slide2"></a>
        <a href="#slide3"></a>
    </div>
</div>
Como você pode ver, nós temos 3 componentes:
  1. 3 elementos auxiliares de contexto
  2. 3 slides
  3. 3 botões pra navegação
E como fazer o slide funcionar? Simples. Primeiro precisamos setar o padrão. E o padrão é:
  1. Todos os botões de contexto são invisíveis o tempo todo.
  2. Todos os slides são invisíveis, exceto o primeiro.
  3. Os botões de navegação devem estar mostrando que o primeiro slide está visível.
Faz sentido não é? Afinal o padrão de um slider é mostrar um slide por vez, e por lógica, o primeiro aparece primeiro :D
Então vamos setar o padrão. 
// Escondendo os botões de contexto
.slider .context-button {
    display: none;
}
// Deixando os slides invisíveis
.slider .slide {
    opacity: 0;
}
// Deixando o primeiro slide visível
.slider .slide:nth-of-type(1) {
    opacity: 1;
}
// Cor padrão dos botões de navegação
.slider .buttons a {
    background-color: #999;
}
// Marcando o primeiro botão de navegação como ativo
.slider .buttons a:nth-child(1) {
    background-color: #fff;
}
Pronto, agora temos nosso padrão setado.
Agora vamos deixar o slider funcionando: 
// Assim que qualquer um dos targets for ativado
// Ou seja, quando existir um #id na url
// Os slides devem se esconder
.slider .context-button:target ~ .slide {
    opacity: 0;
}
// E se o elemento ativado pelo target foi o 1, deixamos o primeiro slide visível
// Se o elemento ativado pelo target foi o 2, deixamos o segundo slide visível
// E assim por diante, pra quantos slides você quiser ter :D
.slider .context-button:nth-child(1):target ~ .slide:nth-of-type(1),
.slider .context-button:nth-child(2):target ~ .slide:nth-of-type(2),
.slider .context-button:nth-child(3):target ~ .slide:nth-of-type(3) {
    opacity: 1;
}
// Fazemos o mesmo com os botões de navegação
// Assim que qualquer um dos targets for ativado
// Os botões de navegação devem ficar todos cinzas
.slider .context-button:target ~ .buttons a {
    background-color: #999;
}
// E se o elemento ativado pelo target foi o 1, deixamos o primeiro botão ativado
// Se o elemento ativado pelo target foi o 2, deixamos o segundo botão ativado
// E assim por diante, pra quantos slides você quiser ter :D
.slider .context-button:nth-child(1):target ~ .buttons a:nth-of-type(1),
.slider .context-button:nth-child(2):target ~ .buttons a:nth-of-type(2),
.slider .context-button:nth-child(3):target ~ .buttons a:nth-of-type(3) {
    background-color: #fff;
}
E pronto!! Nosso slider 100% CSS está pronto (　＾∇＾) yaayyy

A explicação foi feita sem levar consideração nenhum estilo,
Como por exemplo, os slides devem ter position:absolute pra todos ficarem um por cima dos outros.
Os slides devem ter transition:opacity pra que haja efeito de fade entre um slide e outro.
Para ver o css 100% funcional, cheque o estilo do slide no topo da página :D
